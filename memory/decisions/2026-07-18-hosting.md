# 2026-07-18 — Local + ngrok for the video; no serverless hosting

## Decision

Record the demo video against a locally running app exposed via ngrok. Do not deploy to
Vercel/serverless for the demo.

## Context (10 lines max)

- Team chat floated Vercel or GCP hosting so the Gmail add-on can reach `/api/gmail/trigger`.
- The app keeps ALL state in one in-memory `TripObject` singleton with SSE subscribers
  (`core/tripObject.ts`, `core/stream.ts`). Serverless platforms run many short-lived
  instances: state written by one request is invisible to the next, and SSE streams break.
- ngrok gives the add-on a public URL in one command with zero code changes.

## Rejected alternatives

- **Vercel**: fastest deploy, but silently breaks the state model (worst possible failure mid-demo).
- **Cloud Run min-instances=1**: viable later for judges; not needed for a recorded video.

## Revisit when

We need a public URL for live judging rather than a recording.
